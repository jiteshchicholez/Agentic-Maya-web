📚 Core Concepts

Agentic Maya is built around three artifact types: pipelines, policies, and skills. Each piece is designed for predictable, auditable orchestration.

Pipelines

Workflow definitions that chain skills, models, budgets, and policies with checkpointing and completion hooks.

Policies

Conditional governance rules that monitor and enforce safety, cost, PII, and escalation behavior.

Skills

Reusable capabilities like summarization, validation, reporting, and semantic search with governance metadata.

🏛️ Governance Framework

Agentic Maya validates runtime behavior through six enforcement layers, ensuring safe execution at every stage.

Layer Function Control Example
Intent Request validation ACCEPT / REJECT Block malicious instructions
Model Auth Model eligibility ALLOW / BLOCK Prevent unauthorized model use
Budget Token limits CONTINUE / HALT Stop at budget threshold
PII Sensitive data scan ALLOW / BLOCK Block SSN/credit card leaks
External Gate API controls ALLOW / ESCALATE Authorize vendor integration
Escalation Critical workflow AUTO / HUMAN / BOTH Require dual approval

🧠 Memory Tiers

Execution state is separated into session, episodic, long-term, and audit tiers for safe persistence and traceability.

Tier Scope Use Case Lifetime
Session Memory Current run Step results and context During pipeline execution
Episodic Memory Cross-step Checkpoints and resumes Across pauses/rollbacks
Long-Term Memory Multi-session Knowledge, preferences Persistent
Audit Memory Compliance Policy event history Unlimited retention

⌨️ CLI Reference

Use these commands for execution, governance, and recovery.

Run

maya run ./pipelines/document_review.yml

Status

maya status <session_id>

Audit

maya audit <session_id>

Checkpoint

maya checkpoint <session_id> --label after_validation

Rollback

maya rollback <session_id> --to after_validation

Approve

maya approve <session_id> --request-id <id>

💡 Code Details

The documentation site is generated as HTML pages, removing Markdown source files. Core YAML artifacts live in pipelines/, policies/, and skills/.

# pipeline example
pipeline_id: document_review
name: Document Review Pipeline
model: gpt-4-turbo
fallback_model: gpt-3.5-turbo
budget_tokens: 100000
global_policies:
  - maya.audit_all_actions
  - maya.budget_ceiling
  - org.pii_block
  - org.external_call_gate
  - org.critical_escalation
steps:
  - step_id: summarization_step
    skill_id: file_summarizer
    checkpoint_after: true

Each HTML doc page includes detailed content for its artifact class, plus styling optimized for a documentation experience.
