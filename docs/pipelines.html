Pipeline Architecture

Detailed YAML-based pipeline definitions with budgets, governance controls, fallback models, and step checkpointing.

Document Review Pipeline

Automates contract compliance by summarizing, validating, and generating a report with audit logging at every step.

pipeline_id: document_review
name: Document Review Pipeline
model: gpt-4-turbo
fallback_model: gpt-3.5-turbo
budget_tokens: 100000
global_policies:
  - maya.audit_all_actions
  - maya.budget_ceiling
  - org.pii_block
  - org.external_call_gate
  - org.critical_escalation
steps:
  - step_id: summarization_step
    skill_id: file_summarizer
    checkpoint_after: true
  - step_id: validation_step
    skill_id: data_validator
    checkpoint_after: true
  - step_id: report_generation
    skill_id: report_writer
    checkpoint_after: true
Property Value
Budget 100,000 tokens
Governance HIGH
Use case GDPR & SOC2 contract review

Pipeline Execution Model

Each pipeline supports resume, checkpoint, rollback, HITL approval, and final audit summary generation.

RUNNING → checkpoint saved → AWAITING_APPROVAL → resumed after approval → COMPLETED

Use maya status, maya checkpoint, and maya rollback to manage the session lifecycle.
