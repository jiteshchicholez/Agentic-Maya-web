Governance Policy Details

Explore the decision logic, risk levels, and YAML structure for the four core governance policies.

Budget Guard

MEDIUM

Halts execution when token usage exceeds the budget ceiling and escalates to the orchestrator.

policy_id: org.budget_guard
name: Budget Guard
scope: PIPELINE
action: HALT
audit_on_trigger: true
escalation_target: ORCHESTRATOR
governance_level: MEDIUM
cooldown_sec: 0
enabled: true
When it triggers Result
Token usage exceeds ceiling Pause and escalate
Fallback would exceed budget Halt

PII Block

HIGH

Detects sensitive personal data and stops processing until a human approves.

policy_id: org.pii_block
name: PII Detection Block
scope: PIPELINE
action: BLOCK
audit_on_trigger: true
escalation_target: HUMAN
governance_level: MEDIUM
cooldown_sec: 0
enabled: true

Critical Escalation

CRITICAL

Triggers dual escalation when two or more critical violations are detected.

policy_id: org.critical_escalation
name: Critical Escalation
scope: PIPELINE
action: ESCALATE
audit_on_trigger: true
escalation_target: BOTH
governance_level: HIGH
cooldown_sec: 0
enabled: true
